## Naming Conventions

**Files:**
- All lowercase with underscores: `airtable_client.py`, `ollama_client.py`
- Client modules use `_client` suffix: `airtable_client.py`, `ollama_client.py`, `supabase_client.py`
- No `__init__.py` -- flat module structure, not a package

**Functions:**
- Public functions: `snake_case` (e.g., `generate_reels`, `get_client`, `build_generation_prompt`)
- Private/internal functions: `_leading_underscore` (e.g., `_fetch_all`, `_decide_distribution`, `_build_queue_record`)

**Variables:**
- Constants: `UPPER_SNAKE_CASE` (e.g., `BATCH_TYPES`, `SYSTEM_PROMPT`, `TABLE_CLIENTS`)
- Config values: `UPPER_SNAKE_CASE` (e.g., `AIRTABLE_API_KEY`, `OLLAMA_BASE_URL`)
- Local variables: `snake_case`

**Imports:**
- Module aliases used: `import airtable_client as at`, `import ollama_client as ollama`
- Direct imports for framework classes: `from fastapi import FastAPI, HTTPException`

## Where to Add New Code

**New API Endpoint:**
- Add the route handler in `main.py`
- Add Pydantic request/response models in `main.py` (in the `# -- Models` section)
- If the endpoint needs new business logic, add it in `agent.py`

**New Airtable Table Access:**
- Add the table name constant in `config.py` (e.g., `TABLE_NEW_TABLE = "New Table"`)
- Add the query function in `airtable_client.py` following the existing pattern (async function using `_fetch_all` or `_create_record`)
- Use the Airtable formula syntax for filtering (see existing examples in `airtable_client.py`)

**New Prompt Template or Section:**
- Add the template string in `prompts.py`
- Add a `format_*()` function in `prompts.py` to transform raw Airtable data into prompt text
- Wire the new formatter into `build_generation_prompt()` in `prompts.py`

**New LLM Capability (e.g., different generation mode):**
- Add a new function in `ollama_client.py` (following the pattern of `generate()` and `generate_json()`)
- Call it from `agent.py`

**New External Service Integration:**
- Create a new `*_client.py` file at the root level (e.g., `manychat_client.py`)
- Import `config` for credentials and URLs
- Add required env vars to `config.py` and `.env.example`
- Use `httpx.AsyncClient` for HTTP calls (consistent with existing clients)

**New Environment Variable:**
- Add to `config.py` with `os.getenv("VAR_NAME", "default")`
- Add to `.env.example` with a placeholder value
- Add to `.env` with the actual value

**New Business Logic / Pipeline Step:**
- Add the logic in `agent.py`
- If it requires new data, add the data-fetching function in `airtable_client.py`
- If it requires new prompt content, add formatting in `prompts.py`
