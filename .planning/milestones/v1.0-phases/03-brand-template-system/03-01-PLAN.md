<objective>
Add BrandConfig data model, SDMF stage modifiers, multi-font loading, and Airtable brand extraction — threading brand configuration from Airtable through Python to the Remotion Zod schema.

Purpose: Establishes the complete data flow for per-client brand configuration. After this plan, the Python pipeline can fetch a client's brand settings from Airtable, merge with SDMF stage modifiers, and send resolved brand values to Remotion via the HTTP payload. Remotion validates via Zod and loads the correct font.

Output: BrandConfig Pydantic model, brand.py with stage modifiers, extended RenderRequest, Airtable extraction helper, extended Zod schema with BrandConfigSchema, multi-font fonts.ts, and updated RemotionRenderer to pass brand config in the payload.
</objective>
