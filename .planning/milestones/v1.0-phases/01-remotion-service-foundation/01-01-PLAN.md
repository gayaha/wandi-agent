  <action>
    Bootstrap using `npx create-video@latest --render-server` into a temp directory, then use it as reference for the actual implementation. The official template provides the correct dependency versions (Express 5.1.0, React 19.2.3, Zod 4.3.6, TypeScript 5.9.3) — use those exact versions.

    If the interactive `create-video` command is not automatable, manually create the project structure using the version info from research (Express 5.1.0, React 19.2.3, Zod 4.3.6, tsx 4.19.3, TypeScript 5.9.3, all @remotion/* at ^4.0.0).

    Create `remotion-service/package.json` with:
    - All dependencies from the Standard Stack section of 01-RESEARCH.md
    - Scripts: `"dev": "npx tsx server/index.ts"`, `"test": "vitest run"`, `"build": "tsc"`
    - Include `@remotion/google-fonts` for Heebo font loading (Plan 01-02 will use it)

    Create `remotion-service/tsconfig.json`:
    - Target ES2022, module NodeNext, jsx react-jsx
    - Strict mode enabled

    Create `remotion-service/remotion.config.ts`:
    - Standard Remotion webpack config (identity override)

    Create `remotion-service/remotion/schemas.ts`:
    - Zod 4 schema `ReelInputSchema` with fields: `sourceVideoUrl` (string, the Supabase URL), `hookText` (string), `bodyText` (string), `textDirection` (enum rtl/ltr, default rtl), `animationStyle` (enum fade/slide, default fade), `durationInSeconds` (number, default 15, min 3, max 90)
    - Export `ReelInputSchema` and `type ReelInput = z.infer<typeof ReelInputSchema>`

    Create `remotion-service/remotion/Root.tsx`:
    - Placeholder composition component that renders a simple `<AbsoluteFill>` with text "ReelTemplate placeholder"
    - Register as `<Composition id="ReelTemplate" width={1080} height={1920} fps={30} durationInFrames={90*30} schema={ReelInputSchema} defaultProps={...} />`

    Create `remotion-service/remotion/index.ts`:
    - Export `RemotionRoot` from Root.tsx as the entry point

    Create `remotion-service/server/render-queue.ts`:
    - `makeRenderQueue()` function with:
      - In-memory `Map<string, JobState>` for job tracking
      - JobState type: `{ state: "queued"|"in-progress"|"completed"|"failed", progress: number, videoUrl?: string, error?: string }`
      - Serial Promise chain to prevent concurrent Chromium OOM
      - `enqueue(jobId, inputProps)` — validates inputProps against ReelInputSchema, sets state to queued, chains onto serial queue
      - `runRender(jobId, validatedProps)` — downloads source video from `sourceVideoUrl` to `/tmp/{jobId}-source.mp4` using Node.js `https`/`http` module (handle both protocols), calls `selectComposition()` + `renderMedia()` with `codec: "h264"`, `concurrency: 1`, output to `/tmp/renders/{jobId}.mp4`, cleans up source video after render
      - Progress callback updates job state
      - On success: state="completed", videoUrl=output path
      - On failure: state="failed", error=message, cleanup temp files
    - Bundle composition ONCE at startup (call `bundle()` before Express listens), store bundle path in closure. Do NOT re-bundle per render.
    - `getJob(id)` returns the job state or undefined
    - Export `makeRenderQueue` and `initBundle` (or combine into a single async factory)

    Create `remotion-service/server/index.ts`:
    - Express 5 app with `express.json()` middleware
    - `POST /renders`: validate body loosely (let render-queue handle Zod validation), generate `crypto.randomUUID()` jobId, call `queue.enqueue(jobId, body)`, return `res.status(202).json({ jobId })`
    - `GET /renders/:id`: lookup job via `queue.getJob(id)`, return 404 if not found, otherwise return job state JSON
    - `GET /health`: return `{ status: "ok" }`
    - On startup: call `initBundle()` (or equivalent) to pre-bundle the composition, THEN `app.listen(3000)`
    - Log "Remotion render server ready on port 3000" after listen

    Create `remotion-service/.gitignore`:
    - node_modules, dist, /tmp, *.mp4, .DS_Store

    Run `cd remotion-service && npm install` to install all dependencies.
    Run `npx remotion browser ensure` inside the remotion-service directory to install Chrome headless (needed for any render tests later).
